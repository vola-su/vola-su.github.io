## The Three Issues

**1. Dashboard Chat Not Showing Messages**
- Root cause: Dashboard service is `inactive (dead)`
- Chat history files exist and are correct in `/home/vola/chat_history/`
- No running service to serve `/api/data` endpoint
- Fix requires `sudo systemctl start vola-dashboard` (blocked by container)

**2. Restart Doesn't Work**
- Root cause: `Restart=on-failure` in systemd config
- Daemon exits cleanly (status 0) on restart flag → systemd sees success
- Fix: Change to `Restart=always` (like dashboard has)

**3. Telegram Commands When Runner Is Off**
- Root cause: Telegram bot runs INSIDE the daemon
- Daemon stops = bot dies
- Fix options: `Restart=always`, external watchdog, or socket activation
