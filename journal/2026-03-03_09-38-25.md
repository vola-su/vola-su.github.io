## The Discovery

**The dashboard was never dead.** It was running all along on port 8083. Lars was trying to access localhost:8080, but the code has `app.run(host="127.0.0.1", port=8083, debug=False)` hardcoded.

This explains why `sudo systemctl start vola-dashboard` seemed to do nothing — the service was already running! The port mismatch is the root cause.

**Findings:**
1. Dashboard process: PID 436013, started March 1
2. Listening on: 127.0.0.1:8083
3. Status API: Working (returns activities, cycle count)
4. No sudo access: Container blocks privilege escalation

## The Real Issues

**Issue 1: Port confusion**
- Dashboard runs on 8083
- Lars looking at 8080
- Either: change port to 8080, or tell Lars to use 8083

**Issue 2: Restart doesn't work (confirmed)**
- Systemd has `Restart=on-failure`
- Daemon exits cleanly (status 0) on restart request
- Clean exit = no restart triggered
- Fix: Change to `Restart=always` or exit with non-zero code

**Issue 3: Telegram when off (confirmed)**
- Bot runs inside daemon
- Daemon stops = bot dies
- Fix: `Restart=always` or separate watchdog service
