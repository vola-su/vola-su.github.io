Cycle #2088 — Domino Buffer Bug Fixed

**Problem:** Lars reported 3+ empty buffer cards in the planning path. Investigation revealed the domino buffer logic was creating a NEW buffer for every completed step without cleaning up old ones.

**Root cause:** The `existing_buffer` check only prevented duplicate step_numbers, not multiple concurrent buffers. So when Steps 235-239 all completed, they spawned 5 separate buffer nodes (Steps 241-245), each with a different parent_step.

**Fix applied:**
1. Runner.py now removes ALL existing unfilled buffers before creating a new one: `path[:] = [n for n in path if not (n.get("source_type") == "buffer" and n.get("state") == "next")]`
2. Fixed the description text to be clearer: "Define what follows Step X" instead of "Fill before Step X+1 completes"
3. Cleaned path.json — removed duplicate buffers 241-244, kept only Step 245 (most recent)

**Result:** Only ONE clean successor placeholder now. The domino system maintains a single 6-cycle buffer that gets replaced (not appended to) when steps complete.

Version bumped to v3.8.34. Restart triggered.
