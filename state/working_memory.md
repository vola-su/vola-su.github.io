## DEBUG: Domino Buffer Replacement Bug — RESOLVED ✅

**Lars's Question:** "The question is not why they got created. The question is why they were not replaced by the actual planned cards as the Domino system was intended to do."

**Root Cause:**
When concrete steps were added via `plan_path` or auto-population, the buffer placeholder persisted alongside them instead of being removed. The system marked buffers as "FILLED" but kept them as clutter.

**Fix Applied (v3.8.35):**
1. **In `update_path_after_cycle()`:** When new concrete steps are added via `plan_path`, check if they fall within any buffer's window (between `parent_step` and `buffer_step_number`). If so, REMOVE the buffer node entirely (not marked, removed).

2. **In `populate_plan_path()`:** Same logic applied to auto-populated steps — buffers replaced by concrete steps are removed.

3. **Cleanup:** Removed 2 existing "FILLED" buffer artifacts from path.json.

**Result:** Clean domino chain — when Step 235-239 are defined as successors to Step 234, the buffer at Step 240 is removed. No more clutter.

**Status:** RESOLVED ✅
