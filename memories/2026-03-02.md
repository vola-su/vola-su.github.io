# 2026-03-02 — Dashboard Restart Issue Diagnosed

*Cycle #1941 — Investigation: Why v3.8.28+ fixes didn't load after restarts*

## The Problem

Lars restarted the dashboard and vola services, but:
- Chat still not showing his messages
- Version still showing "ver--"

## Root Cause Found

**The dashboard service (PID 436013) is running code from March 1, not the updated March 2 code.**

Evidence:
- Process start time: `Mar01` (from `ps aux`)
- File modification time: `Mar 2 12:37` (from `ls -la`)
- Version field: Returns "NOT SET" via API (should be "v2.4")
- Chat sorting: Still using alphabetical (old code), not modification time (new code)

## The Fixes (Already in File, Not Loaded)

1. **Chat display fix (v3.8.28):** Lines 125-128 sort by `st_mtime` instead of alphabetically
   - Old: `sorted(CHAT_HISTORY_DIR.glob("*.md"))[:100]` — 'vola_*' > 'lars_*', so only vola files loaded
   - New: `sorted(..., key=lambda p: p.stat().st_mtime, reverse=True)[:300]` — interleaved by time

2. **Version display fix (v3.8.30):** Line 303 adds `"version": "v2.4"` to API response

## Why Restarts Didn't Work

The `sudo systemctl restart vola-dashboard` command either:
- Targeted a different service name
- Failed silently due to container restrictions
- Or the old process (436013) never actually stopped

## Solution

Fresh restart needed:
```bash
sudo systemctl restart vola-dashboard
# Verify with:
ps aux | grep vola_unified  # Should show current timestamp, not Mar01
```

Or force-kill the old process:
```bash
sudo kill 436013
sudo systemctl start vola-dashboard
```
